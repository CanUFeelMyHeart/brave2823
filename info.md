Для создания репозитория (инициализация):  
>git init

Вывод текущего состояния репозитория:  
>git status

Добавление файлов к коммиту:
1. git add *name file* (сохраняет файл с введенным именем)
2. git add . (сохраняет все файлы)

Фиксация текущих изменений:  
>git commit -m "*message*"

Вывод всех сохранённых коммитов:  
>git log

* Если логов много нажинаем Enter и открываем все постепенно
* После того как все логи выведены в конце строки появляется END, чтобы вернуться к командной строке нажми Q

Для вывода предыдущих сохранённых файлов (закоммиченых) команда:
>git checkout *name commit*

(вместо полного имени допустимо использовать первые 4 символа коммита)

>git checkout master

(возвращает в последний коммит)

Сравнить записанный коммит с текущим не закомиченым:
>git diff

На этом всё, мы разобрали все команды из текущей лекции.

СПАСИБО ЗА ВНИМАНИЕ

Есть работа с удалёнными репозиториями  
В них есть команды, которые нам пригодяться в работе  
git clone *вставка ссылки* - склонирует репозиторий

git push - протолкнёт ваш репозиторий в эфир гитхаба

git pull - притягивает репозиторий в локальную версию и автоматически мёржит

cd  .\название папки\ - совершает преход в указанную папку
